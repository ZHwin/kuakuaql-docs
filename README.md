## 🚀 使用方法

### 青龙面板（推荐）

```bash
# 1. 安装依赖
依赖管理 → NodeJs → 添加：axios request tough-cookie crypto-js

# 2. 添加订阅
订阅管理 → 新建订阅 → 填入仓库地址

# 3. 配置环境变量
环境变量 → 添加对应脚本的变量

# 4. 运行
定时任务 → 手动运行 或 等待定时执行
```

### 本地运行

```bash
npm install
node <脚本名>.js   # JS 脚本
python <脚本名>.py  # PY 脚本
```

## 🔔 通知功能

所有脚本均使用 `sendNotify.js` 统一推送通知，支持 20+ 种通知方式：

| 渠道 | 环境变量 |
|------|----------|
| Server酱 | `PUSH_KEY` |
| Bark | `BARK_PUSH` |
| PushPlus | `PUSH_PLUS_TOKEN` |
| WxPusher | `WP_APP_TOKEN` |
| 企业微信 | `QYWX_KEY` |
| 钉钉机器人 | `DD_BOT_TOKEN` |
| 飞书机器人 | `FS_BOT_KEY` |
| Telegram Bot | `TG_BOT_TOKEN` + `TG_USER_ID` |
| 邮件推送 | `QQ_MAIL` |

> **青龙面板用户**：直接在环境变量中添加上述配置即可，所有脚本共享。

## ❓ 常见问题

### Q: 报错 `got is not a function`
A: 高版本青龙（Node.js 18+）与 got 库不兼容。脚本已自动使用 axios 替代，只需确保安装了 `axios` 即可。

### Q: 通知发送失败
A: 检查环境变量是否正确配置对应渠道的 Key/Token。

### Q: 青龙拉取订阅 Validation failed
A: 可能是青龙版本问题（v2.20.x 已知 bug），建议降级到 v2.17.x 或检查仓库访问权限。
